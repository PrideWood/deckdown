# Deckdown

一个刻意保持简单的 Markdown → Reveal.js HTML 工具。

## 功能

- 使用标准 Markdown 写作
- 根据 H1/H2/H3 自动组织封面、章节与内容页
- 可选在标题页后自动生成二级标题目录
- 自动目录会同步写入 Markdown，并以原生列表和锚点链接呈现
- 左右切换章节、上下切换章节内内容
- 使用独立一行 `---` 手动分页
- 对长内容进行基础自动分页
- 根据图片、引用等内容自动选择版式
- 四套轻量主题
- PPT 风格功能区，以及字体、字号和 16:9 / 4:3 设置
- 纸张、午夜、海洋、暖调和空白自定义主题
- 可分别覆盖列表、代码框、引用、表格和图片 CSS
- 可按主题编辑全局幻灯片 CSS，并在现有主题样式上继续调整
- `-` 列表始终整组出现，`+` / `*` 列表在开启动画时逐项出现
- 工作台日间/夜间模式及 GitHub 仓库链接
- 实时 Reveal.js 预览
- 移动端采用预览优先布局，可收起编辑区，并通过悬浮按钮在独立页面横屏播放
- 下载一个可离线播放的 HTML 文件
- 导出高清逐页图片 PDF，完整保留浏览器幻灯片样式
- 导出 Deckdown 项目 ZIP，包含 `document.md`、`presentation.html`、
  `project.json` 和统一放在 `assets/` 下的素材
- 从“导入 → 导入 Deckdown 项目”读取 ZIP，恢复文稿、素材、主题和 CSS 设置
- 从“导入 → 导入 HTML”读取新版 Deckdown 独立 HTML，继续编辑其中的文稿、素材和设计设置
- 在演示内打开网页或 HTML 链接，并通过按钮或 Esc 返回
- 通过“插入 → 插入 HTML 文件”读取并嵌入本地 HTML，预览和导出均可使用
- 二级标题后没有三级标题的正文会自动成为无标题居中页
- 选择、粘贴或拖入本地图片，Markdown 使用 `images/` 相对路径
- 光标所在内容与右侧幻灯片自动同步
- 围栏代码块语法高亮和编辑器式代码窗口
- 自动在浏览器本地保存编辑内容

## 开发

```bash
npm install
npm run dev
```

构建：

```bash
npm run build
```

## GitHub Pages 部署

仓库推送到 `main` 分支后，`.github/workflows/deploy-pages.yml` 会自动
构建并发布 `dist`。线上地址为：

```text
https://pridewood.github.io/deckdown/
```

首次部署时，需要在 GitHub 仓库的 **Settings → Pages** 中将
**Source** 设为 **GitHub Actions**。

## Markdown 约定

推荐结构：

```markdown
# 演示标题

副标题

## 第一章

### 第一页

- 内容

### 第二页

更多内容
```

旧文档如果使用“每页一个 H1、页内标题用 H2”，可在“格式化 → 快速整理标题”中自动迁移为 H1 标题页、H2 章节页和 H3 内容页。

本地 HTML 请通过文件选择器插入，Deckdown 会将文件保存到项目的 `html/` 文件夹，或嵌入浏览器素材库。网页打开后，右上角会显示悬浮关闭图标，也可按 `Shift + Esc` 返回幻灯片。

如果文档中存在独立一行的 `---`，则完全按照这些分隔符分页。

图片尺寸可以写在图片名称后：

```markdown
![示例图片|400](images/example.png)
![示例图片|400x300](images/example.png)
```

- 只有图片时，图片自动在正文区居中。
- 同时存在文字和图片时，自动采用左右两栏。
- 只有表格时自动居中；表格与正文同时存在时，根据表格宽高采用上下或左右布局。
- 正文内容默认在标题下方的可用区域内垂直居中。
- 内容溢出时，会自动缩小该页正文，不缩小页面标题。

代码块请标注语言：

````markdown
```css
h1 {
  color: tomato;
}
```
````

需要固定分栏时，每一对 `:::` 表示一列。同一页写一对时内容居中，
写两对时分为两列，写三对时分为三列：

```markdown
:::
左栏内容
:::

:::
右栏内容
:::
```

容器内部无论是否包含空行，都属于同一列，并支持列表、图片、代码、
表格和链接。图片、代码框和表格会在各自列中居中显示。

旧的 `::: block` 开始标记仍然兼容。

## 自定义演示样式

- 修改主题颜色、字体变量：`src/lib/themes.ts`
- 修改标题、列表、图片、代码框和页面布局：`src/lib/exportHtml.ts` 中的 `presentationCss`
- 修改编辑器和工作台界面：`src/App.css`

推荐先复制一个现有主题，再调整这些变量：

- `--slide-bg`：页面背景
- `--slide-fg`：正文颜色
- `--accent`：强调色
- `--accent-soft`：浅色装饰
- `--font-heading`：标题字体
- `--font-body`：正文字体
- `--cover-pattern`：封面背景图案，可使用渐变或 `url(...)`

主题的封面图案由 `.slide-cover` 的 CSS 背景实现，可在
“设计 → 组件 CSS → 全局幻灯片”中按主题独立修改。封面和章节页不再自动添加
伪元素装饰条。

设计菜单还可以分别控制：

- 隐藏或显示 Reveal.js 的进度条与方向箭头。
- 开启自由绘图；播放时点击左下角铅笔或按 `D` 开始批注，点击色点可修改画笔颜色。
- 点击橡皮图标或按 `E` 擦除笔迹，按 `Esc` 退出当前绘图工具。

## 设计原则

1. 原始内容保持为通用 Markdown。
2. 自动判断只做有限、可预测的事情。
3. 输出物是独立 HTML，不依赖 Obsidian 或本地服务器。
4. 不引入 PPT、Marp 和 Slides Extended 兼容层。
