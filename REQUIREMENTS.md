# Deckdown 需求整理

## 一、内容结构

- 一级标题（H1）对应标题页。
- 二级标题（H2）对应章节页，标题在页面中部展示。
- 三级标题（H3）对应内容页标题。
- 二级标题下没有三级标题的正文单独生成无标题页，并水平、垂直居中。
- 可选择是否在标题页之后自动插入目录；目录汇总二、三级标题。
- 提供“快速整理标题”操作，将普通 Markdown 标题归一到上述层级。

## 二、播放与导航

- 左右方向键切换章节。
- 上下方向键切换当前章节内的内容页。
- 移动端以预览为主，编辑区默认收起，并提供独立页面横屏播放入口。
- Markdown 中的网页或 HTML 链接可在演示内打开。
- 支持读取并嵌入本地 HTML 文件。
- 打开的网页提供右上角悬浮退出图标，并支持 Shift + Esc 返回。
- 使用 `+` 或 `*` 开头的列表项可按演示步骤逐项出现，并提供统一开关；其他内容随页面一起出现。

## 三、PPT 风格功能区

- 导入
  - 导入 Markdown
  - 导入 HTML
  - 导入 Deckdown 项目
- 格式化
  - 快速整理标题
- 插入
  - 插入图片
  - 插入网页 / HTML
  - 插入自动目录
- 设计
  - 选择主题
  - 打开字体设置窗口
  - 选择空白自定义主题
  - 开启或关闭图片阴影
- 分别编辑列表、代码框、引用、表格和图片 CSS
- 选择幻灯片大小（16:9、4:3）
- 播放
- 导出
  - 导出独立 HTML
- 恢复
- 右上角工作台夜间模式与 GitHub 仓库链接

下拉菜单按功能分组，预留二级菜单扩展位置，但当前版本不展开复杂的空功能项。

## 四、视觉优化

- 章节标题页居中。
- 引用框缩窄并增加圆角。
- 纯表格居中，混合表格根据形状自动选择上下或左右布局。
- 优化四套主题的选择入口。
- 字体设置窗口分别控制标题字体、正文字体与字号。
- 缩小 Reveal.js 播放控制箭头。

## 五、实现顺序

1. 统一演示设置与 Markdown 标题结构。
2. 生成目录并建立章节/内容的二维导航。
3. 实现网页链接与返回机制。
4. 重构为 PPT 风格功能区。
5. 完成主题、字体、比例和页面视觉优化。
6. 构建、浏览器交互与导出验证。
