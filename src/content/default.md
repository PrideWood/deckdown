# Deckdown

用 Markdown 编写、设计、播放并导出演示文稿。

**MARKDOWN → DECK**

## 认识 Deckdown

### 项目介绍

**Deckdown** 是一款面向浏览器的 Markdown 演示工具。

它将演示内容保留为易读、易修改的 Markdown，同时提供实时预览、主题设计、二维导航和独立导出能力。

你只需要专注于内容，其余工作交给 **Deckdown**。

### 核心能力

- 在左侧编写 Markdown，在右侧实时查看演示效果
- 使用标题层级自动生成章节与内容页
- 自定义主题、字体、颜色和页面比例
- 插入图片、表格、代码、网页和交互式 HTML
- 通过键盘播放、总览和切换幻灯片
- 导出可独立运行的 HTML 演示

### 一份内容，多种使用方式

1. 编写结构清晰的 Markdown
2. 在浏览器中实时检查排版
3. 使用设计面板调整视觉风格
4. 进入播放模式进行演示
5. 导出并分享完整演示文件

### 为什么使用 Markdown

Markdown 让内容保持简单、开放并且容易维护。

**重要信息可以加粗**，*补充说明可以使用斜体*，不再适用的内容可以使用~~删除线~~，技术名称则可以写成 `inline code`。

> 内容应当先具有清晰的结构，然后才需要视觉设计。

## 设计理念

**Deckdown** 不要求你逐页拖动文本框。
它根据 Markdown 的结构理解内容，并将内容转换成适合演示的页面。

## Markdown 编排

### 标题决定页面结构

| Markdown       | 页面作用 | 推荐用途         |
| -------------- | -------- | ---------------- |
| `# 一级标题`   | 标题页   | 演示名称         |
| `## 二级标题`  | 章节页   | 内容分组         |
| `### 三级标题` | 内容页   | 单页主题         |
| `---`          | 手动分页 | 开始新的幻灯片组 |

稳定的三级标题结构可以让源文件与最终演示保持一致。

### 编写内容页

每个三级标题对应一个明确的内容页。

- 标题负责表达这一页的主题
- 正文负责补充背景和说明
- 列表负责呈现并列的信息
- 表格负责比较结构化数据
- 图片负责展示视觉信息

建议每页只表达一个主要观点，避免把多个不同目的的内容挤在同一页。

### 有序步骤

1. 先确定整份演示的主题
2. 使用二级标题划分章节
3. 使用三级标题创建内容页
4. 为每页补充一种主要内容
5. 最后检查播放和导出效果

### 引用版式

> **Deckdown** 的目标不是模仿传统幻灯片编辑器，而是让 Markdown 自然成为一份可以播放、分享和长期维护的演示。

引用适合呈现核心观点、用户评价或需要特别强调的结论。

### 代码窗口

```typescript
const presentation = compileMarkdown(markdown, {
  includeToc: true,
  progressiveReveal: true,
})

exportHtml(presentation)
```

代码块会被展示为独立的代码窗口，适合技术说明、配置示例和接口演示。

### 手动分栏

需要精确控制页面布局时，可以使用成对的 `:::` 创建固定分栏。

```markdown
:::
左栏内容
:::

:::
右栏内容
:::
```

一对 `:::` 会生成一个居中内容区，两对生成两栏，三对生成三栏。

### 自动图文分栏

同一页同时包含正文和图片时，Deckdown 会自动使用图文分栏。

- 文字区域用于解释背景和重点
- 图片区域用于展示产品、图表或示意图
- 页面会根据内容自动分配可用空间

### 打开网页

[访问 DCBJ 数字花园](https://dcbj.online)

网页链接会在演示上方打开，不需要离开当前播放页面。

点击右上角关闭按钮，或按下 **Shift + T**，即可关闭网页并返回幻灯片。

### 插入网页地址

通过“插入 → 插入网页地址”输入网址后，会生成普通的 Markdown 链接：

```markdown
[访问数字花园](https://dcbj.online)
```

以 `http://` 或 `https://` 开头的链接会被识别为网页，并在演示内打开。

### 打开本地交互页面

[打开 Deckdown 交互示例](sample-interactive.html)

本地 HTML 文件也可以覆盖在演示上方，适合展示无需联网运行的交互内容。

### 插入本地 HTML

通过“插入 → 插入 HTML 文件”选择文件后，会生成类似下面的 Markdown：

```markdown
[交互演示](html/demo.html)
```

插入的文件会被保存到项目中或嵌入浏览器素材库，并随导出的演示一起使用。

### 网页与本地 HTML

| 内容类型  | Markdown 地址             | 典型用途             |
| --------- | ------------------------- | -------------------- |
| 在线网页  | `https://...`             | 网站、文档与在线应用 |
| 本地 HTML | `html/demo.html`          | 原型、动画与小游戏   |
| 内置页面  | `sample-interactive.html` | 项目演示与离线内容   |

网页依赖网络连接，本地 HTML 则可以随演示文件一起保存和分享。

## 播放与导出

### 二维导航

- 按 **← / →** 切换章节
- 按 **↑ / ↓** 切换章节内页面
- 按 **F** 进入或退出全屏
- 按 **O** 打开幻灯片总览
- 按 **Shift + T** 关闭网页或交互页面

章节和内容页形成二维结构，让较长的演示仍然容易浏览。

### 逐项出现

使用 `+`、`*` 开头的列表项或有序列表，可以在播放时逐项显示。

1. 当前内容首先保持聚焦
2. 按键后显示下一项内容
3. 演讲者可以自然控制叙述节奏

普通 `-` 列表会整组出现；位于逐项列表之后的文字、图片、表格、视频和代码框会按 Markdown 顺序继续逐帧出现。

如果不需要动画，可以在“设计 → 列表逐项出现动画”中关闭。

### 导出与分享

**Deckdown** 可以将演示导出为独立 HTML 文件。

导出的文件保留：

- 页面结构与主题样式
- 图片、表格和代码内容
- 二维导航与总览模式
- `+` / `*` 列表逐项动画
- 已插入的本地 HTML 页面
